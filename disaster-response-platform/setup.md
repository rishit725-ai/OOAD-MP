# Setup & Run Guide — Disaster Response Coordination Platform

## Prerequisites

| Tool | Version | Download |
|------|---------|----------|
| Java JDK | 17 or later | https://adoptium.net |
| Apache Maven | 3.8+ | https://maven.apache.org/download.cgi |
| Web browser | Any modern | Chrome/Firefox recommended |

Verify installations:
```bash
java -version       # Should show 17.x.x
mvn -version        # Should show 3.8.x or later
```

## Quick Start (3 steps)

> **IMPORTANT**: Run these commands from **Windows PowerShell** or **Command Prompt** (cmd.exe), NOT from WSL/Linux terminal, since the JDK installed is a Windows JDK.

### Step 1 — Open Windows PowerShell and navigate to project root
```powershell
cd C:\Users\rahul\Claude\disaster-response-platform
```

### Step 2 — Build and run
```powershell
mvn spring-boot:run
```

Wait for: `Started DisasterResponseApplication in X seconds`

### Step 3 — Open browser
```
http://localhost:8080
```

**That's it!** The app starts with pre-loaded sample data (6 incidents, 7 teams, 4 shelters, 6 equipment items).

## Demo Flow (Recommended for Evaluation)

### 1. Dashboard Overview
- Open http://localhost:8080
- Show real-time incident counts, team availability, shelter occupancy
- Point out the CRITICAL badge pulsing animation

### 2. Incident Management (Major Use Case 1)
- Click **Incidents** → view the list with colour-coded severity badges
- Click **Report Incident** → fill in: Title = "Earthquake - BTM Layout", Severity = CRITICAL, Location = "BTM Layout", Lat = 12.9100, Long = 77.6100
- Submit → notice the Observer pattern creates a notification and Command Center log entry

### 3. Response Team Management (Major Use Case 2)
- Click **Teams** → show all team types (Medical, Fire, SAR, Police) — all are ResponseTeam subclasses (LSP)
- Click **Add Team** → select "Medical", show type-specific fields appear (JavaScript)
- The Factory Pattern creates the correct MedicalTeam subclass

### 4. Task Assignment with Strategy Pattern (Minor Use Case 1)
- Click **Task Assignment** → show the three strategy cards
- Go to **Assign** tab
- Select the new CRITICAL incident
- Choose **Severity-Based Strategy** → auto-assign
- Show the result: system picked the team with most capacity
- Repeat with **Proximity-Based Strategy** → different team selected (nearest by GPS)
- Repeat with **Resource-Based Strategy** → team with lowest load ratio
- This demonstrates OCP: strategies are interchangeable, no code changed

### 5. Command Center (Minor Use Case 2)
- Click **Command Center**
- Show the **Audit Log** (Command Pattern): every action recorded with type and description
- Show **Notifications** (Observer Pattern): auto-generated by NotificationObserver
- Click **Mark All Read**

### 6. Shelter Management (Major Use Case 3)
- Click **Shelters** → show the card grid with occupancy progress bars
- Open/close a shelter → state changes reflected immediately (State diagram)
- Update occupancy with +10/-10 buttons

### 7. Equipment Management (Major Use Case 4)
- Click **Equipment** → tabular view with status badges
- Release equipment from team → status returns to AVAILABLE

## Project Structure
```
disaster-response-platform/
├── pom.xml                             # Maven build
├── src/main/java/com/disaster/response/
│   ├── DisasterResponseApplication.java
│   ├── model/          # JPA Entities (Incident, ResponseTeam hierarchy, Shelter, Equipment)
│   ├── dto/            # TeamFormDTO (form binding)
│   ├── repository/     # Spring Data JPA interfaces (SRP)
│   ├── strategy/       # Strategy Pattern (SeverityBased, ProximityBased, ResourceBased)
│   ├── observer/       # Observer Pattern (IncidentEventPublisher, observers)
│   ├── command/        # Command Pattern (DisasterCommand, CommandInvoker)
│   ├── factory/        # Factory Pattern (ResponseTeamFactory)
│   ├── facade/         # Facade Pattern (DisasterResponseFacade)
│   ├── service/        # Business logic services (SRP)
│   ├── controller/     # MVC Controllers (LoD)
│   └── config/         # DataInitializer (sample data + observer registration)
├── src/main/resources/
│   ├── application.properties
│   ├── templates/      # Thymeleaf HTML views
│   └── static/css/     # Custom CSS
├── documentation/      # Full project documentation
└── diagrams/           # Mermaid diagram source files
```

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Port 8080 in use | Set `server.port=8081` in application.properties |
| `java.lang.UnsupportedClassVersionError` | Ensure Java 17+ is active: `java -version` |
| `mvn not found` | Add Maven bin/ to PATH |
| Data not loading | Check logs for DataInitializer errors; restart app |
